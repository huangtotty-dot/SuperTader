## 一、回测核心目标

本回测方案从"单一标的信号测试"升级为**"系统级联动测试"**。不仅验证 V1.27 纯净版引擎在华工科技上的表现，更核心的是验证**大盘日线感知（index_regime）与盘中预警（index_regime_intraday）**对个股 T0 信号的"物理刹车"效果。

1. **大盘过滤验证（核心）**：验证在指数暴跌、尾盘跳水（I5）或盘中急跌（I1）时，系统能否成功向华工科技下达"买入封杀令"，避免逆势接飞刀。
2. **纯粹信号验证**：在剥离底仓盈亏（无深套抑制）的前提下，采用**固定仓位（Fixed Size）**，验证系统顺势高抛低吸的纯 Alpha 能力。
3. **防抱死与错杀测试**：验证大盘的 SHARP 锐化分容差机制和个股的 `is_volume_reversal` 机制，是否能在"错杀"行情中敏锐恢复买入通道，避免踏空主升浪。

## 二、回测数据准备与架构拼接

### 1. 数据物料需求

| 项目 | 规格 |
|------|------|
| **个股数据** | 华工科技（000988）1分钟 K线，前复权 |
| **大盘日线** | 上证指数（sh000001）日线（含 amount） |
| **大盘分钟** | 上证指数（sh000001）1分钟 K线 |
| **数据源** | tushare pro `stk_mins`（分钟）+ `daily`（日线） |
| **时间窗口** | 2025-01-01 ~ 2026-07-20 |
| **volume 单位** | 股（非"手"） |
| **权限** | 需 tushare.pro 在线开通分钟权限 |

### 2. 回测调度器管线架构

对于每个交易日（Day T），调度器严格按以下顺序执行，杜绝未来数据：

```
Step 1 (盘前)
  └─ index_regime.py detect(mode="morning")
     └─ 基于 T-1 日收盘的大盘日线 → daily_regime + daily_score + daily_gate
     └─ → 写入 SESSION_CONTEXT

Step 2 (分钟级步进 - 大盘)
  └─ 每分钟 t, 将大盘 09:30~t 的 1分钟K线
     └─ → detect_intraday_alert() → intraday_alerts

Step 3 (分钟级步进 - 个股)
  └─ daily_ctx = {
        "index_regime": 大盘日线态,
        "intraday_alerts": 大盘分时预警,
        "gate_advice": 门控建议,
      }
  └─ signal_engine.evaluate(code, name, df, holding, daily_ctx)
     └─ 读取 intraday_alerts → I1/I5 时锁死买入

Step 4 (订单执行)
  └─ BUY_LOW / SELL_HIGH → 按滑点成交 → 更新冷却时间
```

### 3. 交易假设与固定仓位

| 参数 | 值 | 说明 |
|------|-----|------|
| 初始底仓 | 1000 股 | 确保 T+0 周转 |
| 可用现金 | 50,000 元 | 用于买入 |
| 固定交易单元 | 200 股/笔 | 不论信号强度 |
| 每日最大买入 | 3 次 | max_buy_times = 3 |
| 每日最大卖出 | 3 次 | max_sell_times = 3 |
| 佣金 | 万分之 1.5 | 双边 |
| 印花税 | 千分之 0.5 | 仅卖出 |
| 滑点 | 买入=最高价+0.01, 卖出=最低价-0.01 |

## 五、极端场景追踪验证

### 1. "倾巢之下"测试（盘中跳水拦截）

尾盘 I5 预警触发时，系统必须读取 `intraday_alerts` 并在个股引擎锁死买入。

### 2. "覆巢无完卵"测试（45度阴跌锁死）

大盘 UNI_DOWN 或个股 is_steep_decline 时，切断所有买入通道，保留 SELL_HIGH 能力。

### 3. "无视成本"测试（深套高抛）

浮亏 > 30% 时出现 SPIKE30 或二次冲高未突破，系统果断发出 SELL_HIGH。
