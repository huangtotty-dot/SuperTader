## 1. 模块定位

信号引擎是**做T量化系统的核心决策模块**，负责为每只持仓股票逐分钟计算**买入评分**和**卖出评分**，综合多维度技术指标 + 场景化因子 + 状态机，最终生成交易信号。

### 核心职责

- **逐轮扫描**：每 15 秒对每只持仓执行一次 `evaluate(code, name, df, holding, daily_ctx)`
- **多维度打分**：价格结构、动量、成交量、均线、时间窗口、支撑/压力位
- **场景化因子**：基于 20+ 个历史案例驱动的可观察-确认-锁定状态机
- **信号生成**：当评分超过动态阈值时生成 `Signal` 对象（BUY_LOW / SELL_HIGH / ADD_POS）
- **反T/正T双模式**：根据大盘态势日级决策矩阵切换评分参数

## 2. 核心架构

```
evaluate(code, name, df, holding, daily_ctx)
    │
    ├─ 1. T-mode 解析（正T/反T → 参数切换）
    ├─ 2. 每日状态重置 + 多周期数据加载（1min/5min/15min/日/周/月）
    ├─ 3. 早盘特征计算 + 预警检查（L1~L3 级别）
    ├─ 4. 低点抬高支撑检测（反V转信号）
    ├─ 5. 技术指标聚合（RSI/BB/MACD/EMA/VWAP/MOM/量比）
    ├─ 6. 多分钟周期分析（5min/15min 动能衰竭、支撑、反包）
    ├─ 7. 弱势震荡/45度斜率/均线穿越分类
    ├─ 8. 多维度支撑位识别（昨日低/MA20/MA30/15分低）
    │
    ├─ 买入评分（多因子累加）
    │   ├─ 基础因子：回归空间、RSI超卖、布林偏下、MACD拐头、量能确认
    │   ├─ 分钟线因子：5分大阳线反包、15分动能衰竭、15分强支撑
    │   ├─ 场景化因子：均线回踩、均线上破回踩、尾盘跌破均线
    │   ├─ 支撑因子：MA5/MA10 支撑确认、多维度支撑位
    │   └─ 抑制因子：开盘急跌无反包、弱势震荡禁买、下降趋势禁买
    │
    ├─ 卖出评分（多因子累加）
    │   ├─ 基础因子：回吐空间、RSI超买、布林偏上、MACD拐头、长上影
    │   ├─ 时间因子：持仓时间成熟、收盘前、早盘冲高
    │   ├─ 场景化因子（可观察-确认-锁定）：
    │   │   ├─ GAP_BOUNCE20: 集合竞价弱势 + 早盘反弹高抛
    │   │   ├─ SPIKE30: 开盘冲高回落
    │   │   ├─ FADE15: 冲顶动量衰竭
    │   │   ├─ WEAK_REBOUND_SELL: 低开弱反弹避险
    │   │   ├─ WEAK_REBOUND: 开盘反弹无力/假突破
    │   │   ├─ MA_PRESSURE: 低开冲高均线压制
    │   │   └─ DOUBLE_TOP15: 日内双顶保护
    │   └─ 深套抑制：所有场景化因子在深套时反转（降低而非增加卖出分）
    │
    └─ 最终决策
        ├─ 动态阈值计算（基于市场状态/RSI/量比/日线态）
        ├─ 忽略信号检查（cooldown/redundant/stand_down）
        ├─ 后卖重建锁 + 高点确认延迟
        ├─ 强上涨趋势最终封锁
        └─ Signal 对象生成 / 飞书推送
```

## 3. 状态机群

引擎维护多组状态机，全部存储在 `self` 实例变量中（逐轮扫描间持续）：

| 状态机 | 键 | 用途 |
|--------|------|------|
| 买入冷却 | `self.buy_cooldown` | 买入后 N 分钟内禁止重复买入 |
| 卖出冷却 | `self.sell_cooldown` | 卖出后 N 分钟内禁止重复卖出 |
| 场景因子观察-确认-锁定 | `self.scenario_factor_state` | 每个场景因子走"观察 → 确认 → 锁定 → 冷却"周期 |
| 持仓周期管理 | `self.t_cycle_start_time` | 记录每笔交易的持仓起始时间 |
| 日内高店确认延迟 | `self.peak_tracker` | 防止冲高中途抖动误卖 |
| 卖后等待重建 | `self.post_sell_block_until` | 卖出后 N 分钟内禁止买入 |
| 等接回状态 | `self.awaiting_buyback` | 反T卖出后跟踪是否需要接回 |
| 早盘预警状态 | `self.morning_alert_state` | L1~L3 预警级别 + 纠正跟踪 |
| 信号轨迹追踪 | `self.last_signal_state` | 信号去重（同价位/同方向不重复发） |
| 交易轨迹追踪 | `self.last_trade_state` | 交易执行轨迹 |

### 场景因子状态机（_check_scenario_factor）

```
条件满足 ──观察期──→ 条件持续 ──→ 锁定执行 ──→ 冷却期
    │                     │
    └─条件消失→重置       └─取消条件→重置
```

每个场景因子调用 `_check_scenario_factor(code, factor_name, condition_met, observation_minutes, lock_minutes, cancel_condition)`：

1. **观察期**：条件满足后先观察 N 分钟（防假信号）
2. **确认**：观察期内条件持续存在 → 确认触发，分值生效
3. **锁定**：锁定后 N 分钟内不再重复检查
4. **取消**：取消条件（如价格突破关键位/强势震荡）→ 重置回初始状态

## 5. 动态阈值系统

买入和卖出阈值不是固定值，而是通过 `_dynamic_threshold()` 基于多因素动态计算：

### 5.1 基础阈值

| 参数 | 默认值 |
|------|--------|
| 买入基础阈值 | 35 |
| 卖出基础阈值 | 38 |

### 5.2 影响因素

| 因素 | 买入阈值调整 | 卖出阈值调整 |
|------|------------|------------|
| **市场状态**：dead_water | +10 | ±0 |
| **市场状态**：trend_up | -5（放松） | +10（收紧） |
| **市场状态**：trend_down | +10（收紧） | -5（放松） |

| **日线趋势**：breakdown_risk | +40 | - |
| **日线趋势**：overheated | - | +10 |
| **T模式**：反T模式 | SHORT_MODE_PARAMS 覆盖 | SHORT_MODE_PARAMS 覆盖 |
| **指数门控**：benchmark_gate=weak | +15 | - |
| **指数门控**：benchmark_gate=strong | - | +10 |
| **卖后冷仓**：刚卖出N分钟内 | +20 | - |
| **低点抬高支撑** | -8 | - |
| **大阳线反包确认** | -15 | - |

## 6. 早盘预警系统（V1.25）

基于近两年数据训练的早盘特征分析：

### 预警级别

| 级别 | 含义 | 触发条件 |
|------|------|----------|
| L1 | 早盘弱势 | 低开+首根阴线+量缩 |
| L2 | 开盘急跌 | 5分钟跌幅>2%+量比>1.5+跌破VWAP |
| L3 | 恐慌出逃 | L2条件持续性+跌幅加深+量继续放大 |

### 纠正机制

- 低点抬高支撑确认：10:00 前触发 L2 后，如果出现连续低点抬高 + 价格回到 VWAP 附近，自动降级
- 半日纠正检查：11:30 后检查是否通过逐步回升解除预警
- 纠正确认后推送飞书通知解除

## 7. 信号生成逻辑

### 7.1 决策树

```
                    sell_score ≥ sell_threshold
                    ────┬──── YES
    buy_score ≥ threshold ─┼──── buy_fast_path_gap 条件
         │                 │       └─ SELL_HIGH (卖分显著优于买分)
         │                 │
         │              can_sell && !cooldown
         │                 ├─ 最优卖点绕过检查 → SELL_HIGH
         │                 ├─ 早盘首次卖出保护 → 仅最优卖点可绕过
         │                 └─ 推飞书
         │
    ─────┴──── YES
    BUY_LOW / ADD_POS
    (can_buy && !cooldown
     && 时间>09:40
     && !stand_down ...)
```

### 7.2 Signal 对象

```python
Signal(code, name, action, price, score, reasons, details, indicators, extras)
```

- `action`: BUY_LOW / SELL_HIGH / ADD_POS
- `score`: 最终评分
- `reasons`: 触发原由列表（如["RSI超买","冲顶衰竭"]）
- `details`: 逐因子明细（含指标名/当前值/阈值/解读/加减分）
- `indicators`: 全部技术指标的完整快照

### 7.3 飞书推送保护

| 保护机制 | 逻辑 |
|----------|------|
| 空间冷却 | `_in_cooldown(code, action)` — 同标的同方向 N 分钟内不发 |
| 重复信号过滤 | `_is_redundant_signal()` — 同价位/同方向不重复 |
| 早盘首次卖出保护 | `morning_first_sell_guard` — 10:00 前仅最优卖点可通过 |
| 强上涨最终封锁 | 强势上涨形态中 `sell_threshold += 120` 彻底封锁 |
| 推送节流 | `_should_push(key, signal)` — 同信号 60 分钟不重复推 |

## 8. 正T/反T双模式（V1.26）

### 8.1 T-mode 解析优先级

1. `daily_ctx["t_mode"]`（动态决策注入，最高优先级）

### 8.2 反T模式切换

当 T-mode = `"short"`（反T）时：

```python
p = {**p, **SHORT_MODE_PARAMS}  # 参数覆盖
```

- 反T必须有底仓（`t_qty > 0`）
- 先卖后买
- 卖出门控放松、买入收紧
- 回卖时间窗口放宽

## 9. 市场状态分类（_classify_market_state）

基于当日涨幅、价格 vs VWAP、量比、振幅、EMA 价差：

| 状态 | 含义 | 对阈值影响 |
|------|------|-----------|
| `dead_water` | 波动不足（振幅<0.2%） | 买入阈值+10 |
| `trend_up` | 放量上涨，价>VWAP，EMA向上 | 买入阈值-5，卖出阈值+10 |
| `trend_down` | 放量下跌，价<VWAP，EMA向下 | 买入阈值+10，卖出阈值-5 |
| `bias_up` | 偏强震荡 | 中性 |
| `bias_down` | 偏弱震荡 | 中性 |
| `range_bound` | 窄幅震荡 | 中性 |

## 10. 关键辅助功能

### 10.1 弱势震荡/45度斜率/均线穿越分类（V1.19）

基于 120 分钟窗口的数据驱动分类，用户 07-07 反馈驱动：

- **弱势震荡**：平/低开 + 80%+ 时间低于 VWAP + 今日涨幅 < 1% → 14:30 前绝对禁止买入
- **45度下降+均线上下窜**（华工科技/江丰电子型）：斜率负 + VWAP 穿越 ≥ 2 次 → 可低吸可高抛
- **45度下降+全天均线下**（特变电工/拓维信息型）：斜率负 + 很少穿越 → 不可低吸，保本即跑

### 10.2 低点抬高支撑检测（_check_higher_low_support）

从当日最高点下跌 > 4% 后，检测最近 N 根 K 线的低点是否连续抬高。如确认：
- 清除早盘 L2 预警
- 买入加分 +15，买入阈值 -8
- 华工科技 07-14 案例驱动

### 10.3 持仓周期管理

- `t_cycle_start_time[code]`：记录每笔交易的起始时间
- 正T：买入时开始计时 → 卖出时结束
- 反T：卖出时开始计时 → 买入接回时结束
- 持仓时间不成熟时卖出评分受限

### 10.4 ETF 特殊处理

- **最小交易单位**：100 份
- **早盘卖出观察倍数**：2x（更难触发卖出）
- **阈值封顶**：ETF 买入/卖出阈值 ≤ 38
- **动态份数计算**：`_calc_etf_qty()` — 基于信号强度 + 剩余交易次数 + 可用额度

### 10.5 压力位/支撑位系统（V1.24）

接入 `_calc_ps_levels()` 计算的日线级压力位/支撑位：
- **重要压力**（全部卖出信号）：价格突破关键均线组合（MA5/MA10/MA20/MA30/MA60/MA150）
- **短期压力**（部分卖出）：单一均线压力
- **假突破检测**：突破后回落 → 加强卖出信号

## 11. 冷却与保护机制

| 机制 | 实现 | 默认值 |
|------|------|--------|
| 买入冷却 | 买入后 N 秒禁止重复买入 | 180s |
| 卖出冷却 | 卖出后 N 秒禁止重复卖出 | 120s |
| 卖后重建锁 | 卖出后 N 分钟禁止买入接回 | 10~30分钟（按场景） |
| 冗余信号过滤 | 同价位±0.3% 不重复发信号 | ±0.3% |
| 停摆模式 | 连续亏损/N日无交易 → 提高确认门槛 | - |
| 日线门控 | `daily_buy_t_ok=false` → 买入阈值+25 | 是 |
| 指数门控 | `benchmark_gate=weak` → 买入阈值+15 | 是 |
| 强上涨最终封锁 | 强势上涨形态 → 卖出阈值+120 | 是 |
| 深套抑制 | 亏损 > 5% → 场景化卖出因子反转方向 | 是 |
